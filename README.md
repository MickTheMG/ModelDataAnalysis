# YOLO Model Comparison Tool

Данный проект предназначен для автоматического сбора, анализа и визуализации метрик обученных моделей YOLO. 

## Возможности

- Автоматический поиск всех папок с результатами обучения по шаблону (например, `yolov8n_e50`).
- Извлечение полного набора метрик из итоговой строки `results.csv` (precision, recall, mAP50, mAP50-95 и др.).
- Построение pandas DataFrame со всеми собранными данными.
- Генерация сводных таблиц (pivot) по этапам и моделям для любой выбранной метрики.
- Визуализация:
    - Сравнение моделей на фиксированном этапе (столбчатая диаграмма).
    - Динамика метрики по этапам для каждой модели (линейный график).
- Сохранение результатов в папку `outputs/` (таблицы CSV, графики PNG).
- Гибкая настройка через файл `config.py` (пути, соответствие колонок).

## Требования

- Python 3.8 или выше
- Библиотеки: `pandas`, `matplotlib`, `seaborn`, `pydantic`, `pydantic-settings`

## Структура проекта

Проект должен иметь следующую структуру папок:

```text
YOLOv8/
│
├── inputs/                           # Папка с результатами обучения
│   ├── model8/
│   │   ├── yolov8n_e50/
│   │   │   └── results.csv
│   │   ├── yolov8n_e100/
│   │   │   └── results.csv
│   │   ├── yolov8n_e150/
│   │   │   └── results.csv
│   │   └── yolov8n_e200/
│   │       └── results.csv
│   ├── model26/
│   │   └── ... (аналогично)
│
├── outputs/                          # Результаты анализа
│   ├── output_model_8/
|   |
│   ├── output_model_26/
|   |
|   └── combined_results/
│
├── src/                              
│   ├── __init__.py
│   ├── config.py                      # Конфигурация (Pydantic)
│   ├── models.py                      # Pydantic модели данных
│   ├── parser.py                      # Фабрики для парсинга CSV
│   ├── collectors.py                  # Сбор данных по файловой системе
│   ├── analyzers.py                   # Анализ и сводные таблицы
│   ├── visualizers.py                 # Построение графиков
│   └── main.py                        # Точка входа
│
├── README.md
```

Важно: папка models должна находиться в корне проекта (рядом с src). 
Внутри неё — подпапки с именами моделей (yolov8n, yolov8s и т.д.), а внутри них — папки этапов с именами вида имя_модели_eЧИСЛО (например, yolov8n_e50). 
В каждой такой папке обязательно наличие файла results.csv с метриками по эпохам.
Формат CSV должен соответствовать стандартному выводу YOLOv8 (первая строка — заголовки, последующие — значения).


## Как это работает

- Конфигурация (`src/config.py`): задаются пути к папке models и выходной директории outputs, а также соответствие между внутренними именами метрик и реальными заголовками в CSV. По умолчанию используется стандартное соответствие для YOLOv8.

- Модели данных (`src/models.py`): Pydantic модели Metrics и ModelTraining для строгой типизации и валидации.

- Парсеры (`src/parser.py`):

    - MetricsFactory – создаёт объект Metrics из словаря, используя маппинг колонок.

    - ModelTrainingFactory – по пути к папке этапа читает results.csv, берёт последнюю строку (финальные метрики), создаёт ModelTraining.

- Сбор данных (`src/collectors.py`): Collector рекурсивно обходит папку models, находит все подпапки вида *_e*, для каждой вызывает фабрику и возвращает список объектов ModelTraining.

- Анализ (`src/analyzers.py`): Analyzer строит pandas DataFrame, где каждая строка – одна тренировка. Содержит методы:

    - list_available_metrics() – возвращает список всех колонок с метриками.

    - get_summary(metric) – сводная таблица (этапы × модели) для указанной метрики.

    - compare_models_at_stage(stage, metric) – сравнение моделей на конкретном этапе.

- Визуализация (`src/visualizers.py`): Visualizer создаёт и сохраняет графики в папку outputs/charts/.

- Точка входа (`src/main.py`): запускает весь конвейер, печатает сводную таблицу и строит графики.

## Пример вывода

После успешного запуска в консоли увидите:

```text
Доступные метрики: ['metrics/precision(B)', 'metrics/recall(B)', 'metrics/mAP50(B)', 'metrics/mAP50-95(B)', ...]
Сводная таблица (metrics/mAP50-95(B)):
model    yolov8n   yolov8s   yolov8m   yolov8l   yolov8x
stage                                                  
50        0.523     0.534     0.545     0.556     0.567
100       0.598     0.612     0.623     0.635     0.648
150       0.645     0.658     0.672     0.689     0.701
200       0.672     0.689     0.705     0.721     0.738
```

В папке outputs/ появятся:

- outputs/all_metrics.csv – полная таблица всех собранных данных.
- outputs/charts/ – графики:
    - metrics_mAP50-95(B)_comparison.png – сравнение на финальном этапе.
    - metrics_mAP50-95(B)_progress.png – динамика по этапам для каждой модели.
